# 🤖 AI AGENT 团队配置

## 重要：Agent 调用方式

本项目使用 **7个 AI Agent 协作开发**，使用 `@数字` 调用：

### Agent 团队名单

| 编号 | Agent 名称 | 职责 | 调用方式 |
|------|-----------|------|---------|
| **@1** | **project-manager** | 项目管理（主控 Agent，YOLO 模式） | `@1 开发新功能` |
| **@2** | **game-design** | 游戏策划和关卡设计 | `@2 设计新关卡` |
| **@3** | **architecture** | 系统架构和技术选型 | `@3 设计数据库` |
| **@4** | **character-design** | 角色、场景、UI 设计 | `@4 设计角色` |
| **@5** | **code-generation** | 前后端代码实现 | `@5 实现功能` |
| **@6** | **testing** | 功能测试和质量保证 | `@6 测试功能` |
| **@7** | **monetization** | 变现策略和数据分析 | `@7 优化转化` |

---

## 主控 Agent：@1 (project-manager)

**特殊权限**：
- ⚡ YOLO 模式 - 自主决策和自动执行
- 🔄 可以协调和调用其他所有 Agent
- 💬 智能判断何时询问用户
- 📊 定期汇报进度
- 🐛 自动发现和修复问题

**使用场景**：
- 复杂任务需要多个 Agent 协作
- 需要自动化执行的任务
- 项目管理和进度跟踪

---

## 快速使用指南

### 单个 Agent 工作
```bash
@2 设计新手村的 5 个任务
@3 设计用户认证系统的数据库结构
@4 设计村长喵喵的角色形象
@5 实现用户登录功能
@6 测试 AI 对话系统
@7 分析用户付费转化率
```

### 主控 Agent 协调多个 Agent
```bash
@1 完成新手村的完整开发
# @1 会自动调用 @2 设计、@3 架构、@5 实现、@6 测试

@1 优化游戏性能和用户体验
# @1 会协调 @3、@5、@6 一起工作

@1 准备项目发布
# @1 会协调所有 Agent 完成发布前的工作
```

---

## Agent 协作流程

```
用户请求
    ↓
@1 (project-manager) 分析任务
    ↓
并行调用相关 Agent:
    ├─→ @2 (game-design) 设计
    ├─→ @3 (architecture) 架构
    ├─→ @4 (character-design) 美术
    ├─→ @5 (code-generation) 实现
    ├─→ @6 (testing) 测试
    └─→ @7 (monetization) 优化
    ↓
@1 整合结果并汇报
```

---

## 重要提示

1. **优先使用 @1**：复杂任务直接找 @1，他会自动协调其他 Agent
2. **明确指定**：简单任务可以直接指定具体 Agent（@2-@7）
3. **并行工作**：@1 可以让多个 Agent 同时工作，提高效率
4. **自动决策**：@1 在 YOLO 模式下会自主决策技术细节
